# Alpha - Daily Event Tracker

## Project Overview
Personal markdown-based daily event tracker. Each week gets its own file (`week_[month]_[start_day].md`) with structured notes.

## File Structure
- `week_*.md` — Weekly log files (DO NOT auto-modify content)
- `CHANGELOG.md` — Summarized release notes with sources
- `README.md` — Table of contents and heading structure
- `assets/` — Screenshots, PDFs, and images organized by month
- `LICENSE` — MIT

## Heading Convention (weekly files)
```
# Table of Contents
## [Day] [Month] [Date], [Year]
### News
  - Screenshots, Global News, Semianalysis, TLDR, TBPN
### Work todos:
  - DONE
### TODO:
### Regular todos:
### Action Items and Notes
```

## Rules
- Never modify weekly markdown content without explicit permission
- CHANGELOG.md can contain summarized/enriched versions of events
- Assets go in `assets/[month]/`
- Use websearch to enrich changelog summaries with sources
